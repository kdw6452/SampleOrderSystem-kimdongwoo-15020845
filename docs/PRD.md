# PRD — 반도체 시료 생산 주문 관리 시스템

> 프로젝트 배경 및 기술 아키텍처: [`CLAUDE.md`](../CLAUDE.md) 참조

---

## 1. 이해관계자 (Stakeholders)

| 역할 | 설명 |
|------|------|
| **고객** | 연구소·팹리스·대학 연구실. 시료를 주문하는 외부 주체. 시스템에 직접 접근하지 않음 |
| **주문담당자** | 고객 주문을 콘솔에서 접수·등록·조회. 시료 등록도 담당 |
| **생산담당자** | 접수된 주문의 승인·거절 결정. 생산라인 상태 모니터링 |
| **생산라인** | 웨이퍼 공정 설비. 시료를 한 번에 하나씩 순차 생산. 주문된 시료에 한해서만 가동 |

---

## 2. 시스템 개요

- **운영 방식**: 콘솔 기반. 담당자가 직접 메뉴 번호를 입력하여 명령 수행
- **사용자**: 주문담당자, 생산담당자 (시스템에 직접 접근하는 내부 인원)
- **고객은 시스템에 직접 접근하지 않음** — 주문담당자가 고객 요청을 대리 입력

---

## 3. 생산라인 제약

- 생산라인은 시료를 **한 번에 하나씩** 순차 생산한다.
- **주문이 들어온 시료에 대해서만** 생산을 진행한다 (재고가 충분하면 생산 불필요).
- 하나의 생산라인은 단일 공정 흐름을 가진다 (병렬 생산 없음).

---

## 4. 주문 상태

| 상태 | 설명 | 모니터링 |
|------|------|----------|
| `RESERVED` | 주문 접수 | ✅ |
| `REJECTED` | 주문 거절 — 정상 흐름 외 상태 | ❌ 제외 |
| `PRODUCING` | 주문 승인 완료, 재고 부족으로 생산 중 | ✅ |
| `CONFIRMED` | 주문 승인 완료, 출고 대기 중 | ✅ |
| `RELEASE` | 출고 완료 | ✅ |

### 상태 전이 흐름

```
RESERVED (주문접수)
    ├─ 거절 ──▶ REJECTED                        ← 모니터링 제외
    └─ 승인
         ├─ 재고 충분 ──▶ CONFIRMED ──▶ RELEASE
         └─ 재고 부족 ──▶ PRODUCING ──▶ CONFIRMED ──▶ RELEASE
```

> 전이 규칙 강제 구현: `CLAUDE.md` § 주문 상태 전이 참조

### 흐름별 담당자

| 전이 | 담당자 |
|------|--------|
| → `RESERVED` | 주문담당자 (고객 요청 대리 입력) |
| → `REJECTED` / `PRODUCING` / `CONFIRMED` | 생산담당자 (승인·거절 결정, 재고 확인) |
| `PRODUCING` → `CONFIRMED` | 생산담당자 (ProductionController "생산 완료" 명령 수동 실행) |
| `CONFIRMED` → `RELEASE` | 주문담당자 (출고 처리) |

## 5. 기능 명세

---

### 5-1. 메인 메뉴

- 실행 시 번호 선택 방식의 메뉴 화면을 표시한다.
- 메뉴 진입 전 전체 시료의 요약 정보(시료명·재고)를 함께 표시한다.

| 번호 | 메뉴 | 설명 |
|:----:|------|------|
| 1 | 시료관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 2 | 주문 | 고객 주문 접수 및 생산담당자의 승인·거절 처리 |
| 3 | 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 4 | 출고처리 | CONFIRMED 상태 주문에 대해 출고 실행 |
| 5 | 생산라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |
| 0 | 종료 | 프로그램 종료 |

**AC**
- 목록에 없는 번호 입력 시 오류 메시지를 출력하고 메뉴를 재표시한다.
- 시료가 한 건도 없을 때도 요약 정보 영역은 표시한다(빈 목록).

---

### 5-2. 시료관리

시료(Sample)는 이 시스템의 가장 기본이 되는 단위다. 시스템에 등록된 시료만 주문할 수 있다.

#### 시료 속성

| 속성 | 타입 | 설명 |
|------|:----:|------|
| 시료 ID | int | 시스템 자동 부여, 변경 불가 |
| 이름 | str | 고유값, 공백 불가 |
| 평균 생산시간 | float | 단위: 분. 생산라인 소요 시간 추정에 사용 |
| 수율 | float | 0.0 초과 1.0 이하. 정상 생산 시료 수 / 총 생산 시료 수 |
| 재고 | int | 현재 출고 가능한 시료 수량. 생산 완료 시 증가, 출고 시 감소 |

> 수율 예시: 100개 생산 중 정상 90개 → 수율 = 0.9

#### 시료 등록

이름·평균 생산시간·수율을 입력하면 시료를 등록한다. ID와 초기 재고(0)는 시스템이 자동 부여한다.

**AC**
- 이름이 공백이거나 이미 등록된 이름이면 등록을 거부하고 오류를 출력한다.
- 수율이 0.0 이하이거나 1.0 초과이면 등록을 거부한다. (0.0 허용 시 생산량 계산에서 ZeroDivisionError 발생)
- 평균 생산시간이 0 이하이면 등록을 거부한다.

#### 시료 조회

등록된 전체 시료를 목록으로 표시한다. 각 행에 ID·이름·평균 생산시간·수율·현재 재고를 함께 표시한다.

**AC**
- 등록된 시료가 없으면 "등록된 시료가 없습니다" 메시지를 출력한다.

#### 시료 검색

이름(부분 일치)으로 시료를 검색하여 결과 목록을 표시한다.

**AC**
- 검색 결과가 없으면 "검색 결과가 없습니다" 메시지를 출력한다.
- 검색어가 공백이면 오류를 출력한다.

### 5-3. 주문

고객이 시료를 요청하면 주문담당자가 대신 주문을 생성한다.

> 주문 상태 전이 전체 흐름: §4 주문 상태 참조

#### 시료 예약 (→ RESERVED)

주문담당자가 고객 요청을 받아 시료와 수량을 입력하면 주문이 생성된다.

| 입력값 | 타입 | 조건 |
|--------|:----:|------|
| 시료 ID | int | 등록된 시료의 ID |
| 고객명 | str | 공백 불가 |
| 주문 수량 | int | 1 이상 |

생성된 주문은 즉시 `RESERVED` 상태로 등록된다.

**AC**
- 존재하지 않는 시료 ID 입력 시 오류를 출력하고 주문을 생성하지 않는다.
- 주문 수량이 1 미만이면 오류를 출력한다.
- 고객명이 공백이면 오류를 출력한다.
- 주문 생성 후 부여된 주문 ID와 `RESERVED` 상태를 화면에 표시한다.

#### 접수된 주문 목록

`RESERVED` 상태의 주문만 필터링하여 표시한다.

표시 항목: 주문 ID·시료명·고객명·주문 수량

**AC**
- `RESERVED` 주문이 없으면 "대기 중인 주문이 없습니다" 메시지를 출력한다.

#### 주문 승인 (→ CONFIRMED / PRODUCING)

`RESERVED` 상태의 특정 주문을 승인한다. 현재 재고를 확인하여 두 가지 경로 중 하나로 자동 처리된다.

| 재고 상황 | 처리 결과 |
|-----------|-----------|
| 재고 ≥ 주문 수량 | 즉시 `CONFIRMED` 전환 |
| 재고 < 주문 수량 | 생산라인에 자동 등록 후 `PRODUCING` 전환 |

**AC**
- `RESERVED` 상태가 아닌 주문 ID 입력 시 오류를 출력하고 처리하지 않는다.
- 존재하지 않는 주문 ID 입력 시 오류를 출력한다.
- 승인 후 전환된 상태(`CONFIRMED` 또는 `PRODUCING`)를 화면에 표시한다.
- `PRODUCING` 전환 시 생산라인 등록 결과(부족분, 실생산량)도 함께 표시한다.

#### 주문 거절 (→ REJECTED)

`RESERVED` 상태의 특정 주문을 거절한다. 즉시 `REJECTED` 상태로 전환된다.

> `REJECTED`는 정상 흐름 외 상태로 모니터링에서 제외된다. §4 주문 상태 참조

**AC**
- `RESERVED` 상태가 아닌 주문 ID 입력 시 오류를 출력하고 처리하지 않는다.
- 존재하지 않는 주문 ID 입력 시 오류를 출력한다.
- 거절 후 `REJECTED` 상태로 전환되었음을 화면에 표시한다.

### 5-4. 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 주문 현황과 시료 재고를 함께 표시한다.

> `REJECTED`는 정상 흐름 외 상태로 모니터링 전체에서 제외한다. §4 주문 상태 참조

#### 주문량 확인

유효 상태(`RESERVED` / `PRODUCING` / `CONFIRMED` / `RELEASE`)별 주문 목록을 표시한다.

**AC**
- 각 상태별로 구분하여 표시하며, 해당 상태의 주문이 없으면 해당 항목을 빈 목록으로 표시한다.
- `REJECTED` 주문은 목록에 포함하지 않는다.

#### 재고량 확인

시료별 현재 재고 수량과 주문 대비 재고 상태를 함께 표시한다.

| 재고 상태 | 조건 |
|:---------:|------|
| 여유 | 현재 재고 ≥ 진행 중인 주문 수량 합계 |
| 부족 | 0 < 현재 재고 < 진행 중인 주문 수량 합계 |
| 고갈 | 현재 재고 = 0 |

> 진행 중인 주문: `RESERVED` + `PRODUCING` + `CONFIRMED` 상태 주문의 수량 합계 (동일 시료의 PRODUCING 주문이 여럿이면 전량 합산)  
> `CONFIRMED`를 포함하는 이유: 출고 전까지는 재고를 예약 중인 상태이므로 가용 재고 계산에서 제외해야 한다.

**AC**
- 등록된 시료가 없으면 "등록된 시료가 없습니다" 메시지를 출력한다.
- 재고 상태는 여유·부족·고갈 중 하나로 반드시 표기한다.

---

### 5-5. 출고처리

`CONFIRMED` 상태의 주문에 대해 출고를 실행한다. 출고 시 재고에서 주문 수량만큼 차감되고 주문 상태가 `RELEASE`로 전환된다.

#### 출고 대기 목록

`CONFIRMED` 상태의 주문 목록을 표시한다.

표시 항목: 주문 ID·시료명·고객명·주문 수량·현재 재고

**AC**
- `CONFIRMED` 주문이 없으면 "출고 대기 중인 주문이 없습니다" 메시지를 출력한다.

#### 출고 실행 (→ RELEASE)

특정 주문 ID를 입력하면 해당 주문을 출고 처리한다.

처리 순서:
1. 주문 상태 `CONFIRMED` → `RELEASE` 전환 (실패 시 재고 변동 없이 중단)
2. 전환 성공 후 재고에서 주문 수량 차감

**AC**
- `CONFIRMED` 상태가 아닌 주문 ID 입력 시 오류를 출력하고 처리하지 않는다.
- 존재하지 않는 주문 ID 입력 시 오류를 출력한다.
- 출고 완료 후 차감된 재고 수량과 `RELEASE` 상태 전환 결과를 화면에 표시한다.

---

### 5-6. 생산라인

생산라인 현황과 대기 큐를 표시하고, 생산담당자가 완료 명령을 실행할 수 있다. 주문 승인 시 재고 부족이 확인되면 생산 작업이 자동 등록된다.

> 생산 등록 시점: §5-3 주문 승인 참조  
> `PRODUCING` → `CONFIRMED` 전이는 생산담당자가 "생산 완료" 명령을 수동 실행하여 트리거한다 (비동기 자동 전이 없음).

#### 생산량 계산

수율과 오차(10% 버퍼)를 반영하여 실제 투입 생산량과 예상 소요 시간을 산출한다.

| 항목 | 계산식 | 설명 |
|------|--------|------|
| 부족분 | 주문 수량 − 현재 재고 | 추가로 확보해야 할 수량 |
| 실생산량 | `ceil(부족분 / (수율 × 0.9))` | 수율 손실과 10% 오차 버퍼를 포함한 투입량 |
| 총 생산시간 | `평균 생산시간 × 실생산량` | 단위: 분 |

#### 생산 현황

현재 생산 중인 작업의 정보를 표시한다.

표시 항목: 주문 ID·시료명·고객명·주문 수량·부족분·실생산량·총 생산시간

**AC**
- 현재 생산 중인 작업이 없으면 "생산 중인 작업이 없습니다" 메시지를 출력한다.

#### 생산 완료 명령 (→ CONFIRMED)

현재 생산 중(`PRODUCING`)인 첫 번째 주문을 완료 처리한다. 주문 상태를 `CONFIRMED`로 전이한다.

**AC**
- 현재 생산 중인 작업이 없으면 "생산 중인 작업이 없습니다" 메시지를 출력하고 처리하지 않는다.
- 완료 처리 후 해당 주문 ID와 `CONFIRMED` 전환 결과를 화면에 표시한다.
- 재고는 주문 승인 시점에 계산·저장된 `shortfall` 값만큼 증가한다 (`stock += order.shortfall`). 실제 생산량(`actual_qty`)의 잉여분은 재고에 반영하지 않는다.

#### 대기 주문 확인

생산 큐에서 대기 중인 작업 목록을 FIFO 순서로 표시한다.

표시 항목: 대기 순번·주문 ID·시료명·고객명·주문 수량·실생산량·총 생산시간

**AC**
- 대기 중인 작업이 없으면 "대기 중인 생산 작업이 없습니다" 메시지를 출력한다.
- 목록은 생산라인에 등록된 순서(FIFO)로 출력한다.

## 6. 기능 범위

### In-Scope

- 시료 등록·목록 조회·검색
- 주문 접수·승인·거절
- 생산라인 현황 조회 (생산 중 / 대기 큐)
- 출고 처리
- 재고 및 주문 상태 모니터링

### Out-of-Scope

- 고객이 직접 접근하는 UI (웹·앱)
- 네트워크 통신, REST API
- 인증·권한 관리
- 복수 생산라인 병렬 운영
- 비동기(asyncio) 처리
